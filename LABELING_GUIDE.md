## 1. 시작하기

### 1.1 웹 애플리케이션 접속

웹 브라우저에서 제공된 URL로 접속합니다:
- 프론트엔드: `http://localhost:3000`
- 백엔드 API: `http://localhost:8000`

### 1.2 메인 화면 구성

메인 페이지(Pipeline 대시보드)에서는 현재 레이블링 진행 상황을 확인할 수 있습니다:

| 메뉴 | 설명 |
|------|------|
| **Pipeline** (/) | 전체 워크플로우 상태 대시보드 |
| **Gallery** (/gallery) | 이미지 갤러리 및 레이블링 작업 공간 |
| **Upload** (/upload) | 이미지 및 클래스 파일 업로드 |
| **Training** (/training) | 모델 학습 (관리자용) |
| **Models** (/models) | 학습된 모델 목록 (관리자용) |

## 2. 이미지 업로드

### 2.1 Upload 페이지 접속

상단 메뉴에서 **Upload** 또는 `/upload` 경로로 이동합니다.

### 2.2 이미지 업로드 방법

1. **드래그 앤 드롭**: 이미지 파일을 업로드 영역으로 끌어다 놓습니다.
2. **클릭하여 선택**: 업로드 영역을 클릭하고 파일 탐색기에서 이미지를 선택합니다.

### 2.3 지원 파일 형식

- **이미지**: JPG, JPEG, PNG, BMP, WEBP
- **한 번에 여러 이미지** 업로드 가능

### 2.4 클래스 정의 파일 업로드 (선택사항)

레이블링에 사용할 클래스 목록이 담긴 텍스트 파일을 업로드할 수 있습니다:
```
person
car
dog
cat
...
```

## 3. 갤러리에서 이미지 선택

### 3.1 Gallery 페이지 접속

상단 메뉴에서 **Gallery** 또는 `/gallery` 경로로 이동합니다.

### 3.2 이미지 필터링

갤러리 상단의 필터 탭을 사용하여 이미지를 분류할 수 있습니다:

| 필터 | 설명 |
|------|------|
| **All Images** | 모든 이미지 표시 |
| **Labeled** | 레이블링 완료된 이미지 |
| **Unlabeled** | 아직 레이블링되지 않은 이미지 |
| **Needs Review** | 검토가 필요한 이미지 (자동 레이블링 후 confidence가 낮은 경우) |

### 3.3 이미지 선택

레이블링할 이미지를 **클릭**하면 오른쪽에 레이블링 워크스페이스가 열립니다.

## 4. 레이블링 워크스페이스 사용법

### 4.1 워크스페이스 구성

```
┌─────────────────────────────────────────────────────────────┐
│  [도구 모음]                                                 │
│  ┌─────────────────────────────────────────────────────────┐│
│  │                                                         ││
│  │                    이미지 캔버스                         ││
│  │          (마스크/폴리곤이 오버레이됨)                    ││
│  │                                                         ││
│  └─────────────────────────────────────────────────────────┘│
│  [어노테이션 목록]                          [클래스 목록]    │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 도구 모음 버튼

| 버튼 | 기능 | 설명 |
|------|------|------|
| **HILIPS** | AI 자동 레이블링 | Gemini LLM + SAM2를 사용한 완전 자동 레이블링 |
| **SAM v2** | 클릭 기반 세그멘테이션 | 객체를 클릭하면 자동으로 마스크 생성 |
| **Manual** | 수동 폴리곤 | 직접 폴리곤 점을 찍어 영역 지정 |
| **Zoom In/Out** | 확대/축소 | 이미지 확대 및 축소 |
| **Pan** | 이동 | 확대된 이미지 이동 |
| **Undo/Redo** | 실행 취소/다시 실행 | 작업 취소 및 복원 |

### 4.3 확대/축소 및 이동

- **마우스 휠**: 확대/축소
- **마우스 드래그** (Pan 모드): 이미지 이동
- **확대/축소 버튼**: 도구 모음의 +/- 버튼 사용

## 5. 레이블링 방법

### 5.1 HILIPS 자동 레이블링 (권장)

가장 빠르고 효율적인 방법입니다.

**사용 방법:**
1. 도구 모음에서 **HILIPS** 버튼을 클릭합니다.
2. AI가 자동으로 이미지를 분석합니다:
   - **Gemini LLM**: 객체 탐지 및 클래스 식별
   - **SAM2**: 정밀 세그멘테이션 마스크 생성
3. 생성된 어노테이션이 이미지 위에 오버레이됩니다.
4. 결과를 확인하고 필요시 수정합니다.

**Confidence Score:**
- 각 어노테이션에는 신뢰도 점수가 표시됩니다.
- **0.8 이상**: 높은 신뢰도 (녹색)
- **0.5 ~ 0.8**: 중간 신뢰도 (노란색) - 검토 권장
- **0.5 미만**: 낮은 신뢰도 (빨간색) - 반드시 검토 필요

---

### 5.2 SAM v2 클릭 기반 세그멘테이션

특정 객체만 정밀하게 세그멘테이션하고 싶을 때 사용합니다.

**사용 방법:**
1. 도구 모음에서 **SAM v2** 버튼을 클릭합니다.
2. 세그멘테이션할 객체의 **내부를 클릭**합니다.
3. SAM2가 클릭한 지점을 기준으로 객체 마스크를 생성합니다.
4. 오른쪽 클래스 목록에서 **클래스를 선택**합니다.
5. 필요시 추가 클릭으로 마스크를 조정합니다:
   - **좌클릭**: 마스크에 포함할 영역 추가
   - **우클릭**: 마스크에서 제외할 영역 지정

**팁:**
- 객체의 중심부를 클릭하면 더 정확한 결과를 얻을 수 있습니다.
- 여러 번 클릭하여 마스크를 세밀하게 조정할 수 있습니다.

---

### 5.3 수동 폴리곤 그리기

AI가 인식하지 못하는 객체나 정밀한 수동 레이블링이 필요할 때 사용합니다.

**사용 방법:**
1. 도구 모음에서 **Manual** 버튼을 클릭합니다.
2. 객체의 외곽선을 따라 **점을 클릭**하여 폴리곤을 그립니다.
3. 마지막 점에서 **첫 번째 점을 다시 클릭**하거나 **Enter** 키를 눌러 폴리곤을 완성합니다.
4. 오른쪽 클래스 목록에서 **클래스를 선택**합니다.

**폴리곤 편집:**
- **점 이동**: 폴리곤 점을 드래그하여 위치 조정
- **점 추가**: 폴리곤 선 위를 더블클릭
- **점 삭제**: 점을 선택 후 Delete 키

## 6. 어노테이션 편집 및 수정

### 6.1 어노테이션 선택

- **캔버스에서 클릭**: 이미지 위의 마스크/폴리곤을 클릭하여 선택
- **어노테이션 목록에서 클릭**: 왼쪽 하단의 어노테이션 목록에서 선택

### 6.2 클래스 변경

1. 수정할 어노테이션을 선택합니다.
2. 오른쪽 클래스 목록에서 새로운 클래스를 클릭합니다.

### 6.3 어노테이션 삭제

1. 삭제할 어노테이션을 선택합니다.
2. **Delete** 키를 누르거나 어노테이션 목록의 **휴지통 아이콘**을 클릭합니다.

### 6.4 어노테이션 속성 수정

어노테이션 목록에서 각 어노테이션의 속성을 확인하고 수정할 수 있습니다:
- **클래스 이름**
- **Confidence 점수** (자동 레이블링의 경우)
- **영역 크기**

## 7. 저장 및 내보내기

### 7.1 저장하기

레이블링 작업을 완료한 후 반드시 저장해야 합니다.

**저장 방법:**
1. 도구 모음의 **Save** 버튼을 클릭합니다.
2. 또는 키보드 단축키 **Ctrl + S** (Windows) / **Cmd + S** (Mac)를 사용합니다.

### 7.2 저장 형식

어노테이션은 **COCO 형식**으로 저장됩니다:
```json
{
  "image_id": "image_001",
  "annotations": [
    {
      "id": 1,
      "category": "person",
      "segmentation": [[x1, y1, x2, y2, ...]],
      "bbox": [x, y, width, height],
      "area": 12345,
      "confidence": 0.95
    }
  ]
}
```

### 7.3 자동 저장

- 시스템은 주기적으로 작업 내용을 자동 저장합니다.
- 브라우저를 닫아도 작업 내용이 유지됩니다.

## 8. 검토가 필요한 이미지 처리

### 8.1 Needs Review 필터 사용

배치 자동 레이블링 후 신뢰도가 낮은 이미지는 자동으로 "Needs Review" 상태로 분류됩니다.

**검토 프로세스:**
1. 갤러리에서 **Needs Review** 필터 탭을 클릭합니다.
2. 검토가 필요한 이미지 목록이 표시됩니다.
3. 각 이미지를 클릭하여 어노테이션을 확인합니다.
4. 필요시 수정합니다:
   - 잘못된 클래스 → 올바른 클래스로 변경
   - 부정확한 마스크 → 수동으로 조정
   - 누락된 객체 → 추가 레이블링
   - 잘못된 어노테이션 → 삭제
5. 수정 완료 후 **Save**를 클릭합니다.
6. 이미지가 자동으로 "Labeled" 상태로 변경됩니다.

### 8.2 우선순위

검토 대기열은 다음 우선순위로 정렬됩니다:
1. Confidence가 매우 낮은 어노테이션이 있는 이미지
2. 이상치(비정상적인) 탐지 수를 가진 이미지
3. 오래된 이미지

## 9. 자주 묻는 질문

### Q1. HILIPS 버튼을 눌렀는데 아무 반응이 없어요.
**A:** 백엔드 서버가 실행 중인지 확인하세요. 또한 이미지가 제대로 로드되었는지 확인하세요.

### Q2. SAM v2 클릭이 작동하지 않아요.
**A:** SAM v2 모드가 활성화되어 있는지 확인하세요. 버튼이 활성화 상태(강조 표시)여야 합니다.

### Q3. 폴리곤이 너무 복잡해요. 간단하게 만들 수 있나요?
**A:** 가능한 한 적은 수의 점으로 객체의 외곽선을 따라가세요. 곡선 부분에서만 더 많은 점을 사용하세요.

### Q4. 저장했는데 다음에 다시 열면 어노테이션이 없어요.
**A:** 저장 버튼을 클릭한 후 "저장 완료" 메시지가 표시되었는지 확인하세요. 네트워크 오류가 있었을 수 있습니다.

### Q5. 어떤 클래스를 선택해야 할지 모르겠어요.
**A:** 제공된 클래스 정의 문서를 참조하세요. 확실하지 않은 경우 가장 유사한 클래스를 선택하고, 추후 검토 과정에서 수정할 수 있습니다.

### Q6. 실수로 어노테이션을 삭제했어요.
**A:** **Ctrl + Z** (Windows) / **Cmd + Z** (Mac)를 눌러 실행 취소하세요. 또는 도구 모음의 Undo 버튼을 클릭하세요.

### Q7. 이미지가 너무 어두워서 객체를 구분하기 어려워요.
**A:** 확대 기능을 사용하거나, 브라우저의 밝기 조절 확장 프로그램을 사용해 보세요.

### Q8. 한 이미지에 같은 클래스의 객체가 여러 개 있어요.
**A:** 각 객체마다 별도의 어노테이션을 생성해야 합니다. 하나의 어노테이션에 여러 객체를 포함시키지 마세요.
